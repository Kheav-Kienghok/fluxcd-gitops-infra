# FluxCD GitOps Infrastructure

Provisions an AWS EC2 instance (`t3.medium`, Ubuntu 24.04 LTS) and configures it as a local Kubernetes lab using Terraform and Ansible.

## What gets deployed

| Layer | Tool | Purpose |
|-------|------|---------|
| Infrastructure | Terraform | EC2 instance, SSH key pair, security group |
| Configuration | Ansible | Docker, kubectl, Kind, Flux CLI |

**Ansible installs:**
- Docker CE + Compose plugin
- kubectl v1.30.2
- Kind v0.23.0 (Kubernetes in Docker)
- Flux CLI (latest)

## Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.5
- [Ansible](https://docs.ansible.com/ansible/latest/installation_guide/) >= 2.14
- AWS credentials configured (`aws configure` or environment variables)

## Project structure

```
.
├── Makefile              # One-command deployment
├── terraform/
│   ├── provider.tf       # AWS, TLS, local providers
│   ├── variables.tf      # Region, instance type, key name
│   ├── main.tf           # EC2, key pair, security group, inventory output
│   └── outputs.tf        # ec2_url, ssh_command
├── ansible/
│   ├── ansible.cfg       # SSH settings, key path
│   ├── inventory.ini     # Auto-generated by Terraform
│   └── playbook.yml      # Install Docker, Kind, Flux CLI
└── secret/
    └── ec2_key.pem       # SSH private key (git-ignored, written by Terraform)
```

## Usage

### Full deployment (one command)

```bash
make deploy
```

This runs `terraform init` → `terraform apply` (auto-approve) → `ansible-playbook` in sequence.

### Step by step

```bash
make init      # Initialise Terraform providers
make apply     # Provision AWS resources (prompts for confirmation)
make ansible   # Run Ansible playbook
```

### Teardown

```bash
make destroy
```

## Outputs

After `terraform apply`, the following are available:

```bash
cd terraform && terraform output
```

| Output | Example |
|--------|---------|
| `ec2_url` | `http://1.2.3.4` |
| `ssh_command` | `ssh -i ../secret/ec2_key.pem ubuntu@1.2.3.4` |
| `ec2_public_ip` | `1.2.3.4` |

The `ansible/inventory.ini` file is also written automatically by Terraform.

## Notes

- The SSH private key is stored at `secret/ec2_key.pem` (mode `0400`) and is git-ignored.
- Ansible waits for SSH availability and cloud-init to finish before installing packages.
- Security group allows inbound SSH (22), HTTP (80), and HTTPS (443) from anywhere. Restrict `cidr_blocks` in `terraform/main.tf` for production use.
